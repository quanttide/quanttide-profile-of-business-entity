# 新媒体运营笔试题

考核项目：量潮课堂 · 新媒体运营
考核时长：2-3天，完成后提交
提交物：一份改写稿 + 改写思路说明 + 简要规划
工作环境：远程异步

## 一、背景说明

### 我们做什么

量潮课堂教开发者在 AI 时代怎么写出高质量代码。

我们不教怎么写代码——市面上教这个的太多了。我们教的是：当 AI 写了一万行代码、系统开始失控的时候，你怎么把局面拉回来。

### 我们的教学理念

我们内部验证过一件事：十年老手的工作体系，可以在一周内成套传递给新人。

一个大一文科生，学了七天 Vibe Coding，开始独立设计产品，甚至推翻了我的技术方案，提出了更优雅的架构设计。这个过程已经记录成文。

我们管这套方法叫"开全图"——用精良的示范和精准的解释，让新人在实战中快速抓住核心意图。

### 我们的内容风格

我们在公众号和小红书发布内容，已验证过一个模式：真实的技术洞察 + 诚实的挣扎故事 + 不卖课的姿态 = 最有效的转化。

### 你的任务

我们有内部经验文章，需要你把它改写成能吸引目标用户、推动课程咨询的商业内容。

## 二、笔试任务

### 任务一：学习——分析一篇高转化内容

阅读《重构是 Vibe Coding 时代的必备技能》这篇文章，回答：

1. 这篇文章的每一段分别完成了什么任务？请列出结构大纲并标注每段作用。
2. 哪几个点推动了读者的信任和咨询意愿？为什么有效？
3. 这篇文章克制了什么没写？为什么这种克制是有效的？
4. 提炼出 3 条可复用的内容创作原则。

### 任务二：实践——改写一篇内部经验文章

请将《如何快速向新人成套传递十年老手的工作体系？》改写为一篇面向外部读者的商业内容。

要求：

· 保留核心故事（新人一周推翻老手方案）
· 用"事故故事"叙事——展示真实的挣扎和转折
· 读者读完能产生两个感受："这说的就是我的问题"、"我想知道他们怎么做到的"
· 不要写成课程广告，课程信息只在最自然的节点出现

提交时附上改写思路说明：

· 原文中你保留了什么？为什么？
· 你删减或调整了什么？为什么？
· 在哪些地方做了商业化处理？如何服务于转化？

### 任务三：延伸——简要规划

以这篇改写后的文章为起点：

1. 如果再写 2 篇形成系列，你选什么选题？每篇的"冲突锚点"是什么？
2. 从"读者读到文章"到"成为课程咨询用户"，转化路径怎么设计？


## 三、提交要求

提交形式：一份 Markdown 文档。

内容结构：

· 任务一：文章拆解 + 创作原则提炼
· 任务二：改写后的完整文章 + 改写思路说明
· 任务三：系列选题规划 + 转化路径设计

评估维度：

· 改写能力：能否将内部经验转化为有转化力的商业内容
· 叙事意识：能否用"真实挣扎故事"构建信任，而非堆砌卖点
· 转化设计：能否让读者自然产生"我想学"的冲动
· 用户视角：是否始终在想"读者读到这会怎么想"


## 四、关于工作方式

· 异步协作，文档优先
· 每个决策说明"为什么"
· 鼓励在过程中提问


期待看到你的改写稿。
