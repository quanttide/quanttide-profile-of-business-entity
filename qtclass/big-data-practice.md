# 大数据应用项目实践

基于电商数据的中国高频价格指数计算。

## 项目背景

消费者价格指数（CPI）是衡量通胀水平的重要宏观经济指标，但传统 CPI 依赖人工调研与月度统计更新，存在时效性差、颗粒度粗、覆盖不全等固有缺陷。

随着电商平台数据的大规模生成，构建高频（按日）价格指数成为可能。电商数据具备更新快、覆盖面广、精度高等优势，可突破传统 CPI 的限制，形成对市场更灵敏的监测手段。

本实践项目源于量潮科技前身 —— 厦门大学 China's Prices Project（CPP）课题组十年前的积累。CPP 参考 MIT Billion Prices Project 和 Cavallo 的研究，基于中国官方 CPI 分类标准，每天采集天猫和京东平台 1000 余个关键词、约 500 万数据点，编制中国高频线上价格指数。CPP 经历了从个人电脑爬虫到腾讯云服务器集群、从 13 人团队手动维护到自动化系统的技术演进，这些积累沉淀为本实践项目的课程设计基础。

十年后的今天，本项目采用 2026 年的最新技术栈重新实现这一经典课题：以对象存储替代本地文件、ClickHouse 替代 MongoDB、Python 自动化流水线替代手动运维，打造一套现代化、高可用的高频价格指数计算方案。

## 实践目标

- 熟练掌握对象存储的数据管理能力
- 利用 ClickHouse 实现高效的列式查询与价格指数构建
- 基于 Python 的自动化计算流程，实现一套自动化、高并发、可复用的价格指数计算体系
- 使用数据湖的解决方案（可选，附加题）

## 架构设计

整合项目整体思路，包括存储、计算、可视化三个部分。

### 数据预处理

- 将数据集上传至对象存储，构建个人 Bucket 并配置权限
- 对接对象存储与 ClickHouse
- 对字段类型进行适配与转换，统一时间格式、价格单位、分类编码

### 价格指数核心算法

**1. 异常处理机制**（ClickHouse 实现）

价格可能为空或存在异常值，需要在计算前进行清洗和过滤。

**2. 指数计算（支持链式）**

- 设计分类价格指数：每类商品按销量加权平均价格
- 构建日价格指数时间序列，结合 GROUP BY 与 ARRAY JOIN 聚合
- 链式指数计算：当期指数基于上一期指数计算，通过环比连接保持指数连续性

### 可视化

采用 BI 工具或 Python matplotlib 等进行可视化，输出内容为日度指数趋势图。
