# 公司代表大会议事章程

## 背景

本章程旨在明确公司代表大会的运作机制，确保会议效率和决策质量。会议的议事效率高度依赖书记处提前整理各项事务。

## 会议体系架构

### 总体结构

```
┌─────────────────────────────┐
│      公司代表大会            │  ← 所有人通过确认
└──────────────┬──────────────┘
               ↑
    ┌──────────┴──────────┐
    │        书记处        │  ← 整理汇总、统筹协调
    └──────────┬──────────┘
               ↑
    ┌────────┬────────┬────────┐
    │        │        │        │
管理会   技术会   办公会   周会   ← 日常议事
```

### 会议层级

| 层级 | 会议 | 说明 |
|------|------|------|
| L1 | 公司代表大会 | 最终决策，所有人通过确认 |
| L2 | 书记处 | 整理汇总，统筹协调 |
| L3 | 小会 | 管理会、技术会、办公会、周会 |

## 会议类型

### 1. 秘书处会议

离职手续完成后，执行委和技术委将退化回秘书处。秘书处召开两个例会：

| 会议 | 主持 | 参与 | 讨论内容 |
|------|------|------|----------|
| 管理会 | 秘书长 | 部门负责人 | 管理事务 |
| 技术会 | CTO | 技术负责人 | 技术事务 |

**技术会特殊安排**：
- CEO 与 CTO 办公会共同召开技术会

### 2. 书记处会议

书记处由雅秘书长负责组织，精简人员后设置四个会议：

- 办公会
- 周会
-（及其他日常会议）

### 3. 扩大会议机制

根据需要吸纳各部门负责人参与讨论：

| 场景 | 扩大形式 |
|------|----------|
| 需要部门负责人参与 | 管理会 → 执行委会议 |
| 需要技术负责人参与 | 技术会 → 技术委会议 |

## 运作流程

### 会议循环

```
┌──────────────┐    发现问题     ┌──────────────┐
│   大会       │ ──────────────→│    小会      │
│              │ ←──────────────│              │
└──────────────┘   持续循环     └──────────────┘
     ↑                              │
     └──────── 讨论汇总后上会 ──────┘
```

### 管理会运作

每周精选 1-2 件需要展开讨论的核心事项（大事或一组大事），深入分析原因，形成提案提交至周会。

**示例**：
> 因浙理工款项未到导致公司现金流连锁反应，管理会专门讨论各方面原因及整改方案，最终以提案形式提交周会。

### 书记处工作

- 接收原办公会内容，每周定时过一遍
- 了解全局，统筹协调
- 提前整理会议材料，确保大会效率

## 新人培养机制

作为新人培养方式，书记处轮值制度安排如下：

1. **参与范围**：所有轮值书记
2. **培养对象**：有培养空间的人员
3. **职责**：轮值人员根据讨论结论负责整理

**SSC 成员特别安排**：
- 侧重参与书记处工作
- 通过书记处了解全局

## 会议效率保障

若书记处未提前整理到位，可能导致：
- 开会效率低
- 无法形成有效决议
- 会议流于形式

因此，书记处的准备工作是大会质量的必要保障。
